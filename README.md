# Digital Footprint Manager

> **"인터넷에 남긴 나의 흔적을 찾아 정리하는 디지털 청소 서비스"**

## 1. 문제의식

사람들은 수많은 온라인 서비스에 가입하지만, 시간이 지나면 자신의 디지털 흔적을 관리하지 못한다.

- 오래된 계정을 기억하지 못함
- 사용하지 않는 계정이 방치됨
- 과거 게시물·공개 정보 관리 어려움
- 개인정보 유출 위험을 파악하기 어려움

**핵심 문제**

> "내가 인터넷에 무엇을 남겼는지 모르고, 정리 방법도 어렵다."

## 2. Target User

### ① 20~30대 디지털 사용자

- SNS, 쇼핑몰, 커뮤니티 등 다양한 서비스 이용
- 오래된 계정 정리 필요

### ② 개인정보 보호 관심 사용자

- 불필요한 계정 삭제 및 데이터 관리 필요

## 3. Solution

사용자의 디지털 흔적을 탐색하고 정리 방향을 제안한다.

### ① 계정 탐색

- Gmail API — 가입·인증·결제 메일의 흔적에서 계정 후보 추출
- 스캔은 브라우저 안에서 수행하고, 메일 원문·본문·제목은 서버로 보내지 않는다

→ **"내가 가입한 사이트 후보 목록" 제공**

### ② 위험도 분석

- 장기간 미사용 계정
- 개인정보 저장 가능성 (서비스 카테고리 기반)
- 알려진 유출 연관 여부

### ③ 삭제 지원

- 탈퇴 링크·가이드 제공
- 개인정보 삭제 요청문 생성
- 삭제 우선순위 추천

## 4. MVP — 웹서비스

설치 없이 브라우저에서:

1. Gmail 연결 (사용자가 버튼을 눌렀을 때만 권한 요청)
2. 가입·인증 흔적 탐색
3. 정리 필요 계정 추천

**결과 예시**

> 계정 후보 63개 발견
>
> 8개 계정 정리 추천
>
> 3개 계정 알려진 유출 연관

### MVP 기능

1. Gmail에서 가입/인증 메일 찾기
2. 서비스별 리스트 만들기
3. 마지막 관련 메일 날짜 표시
4. 1년 이상 미사용 후보 추천
5. 탈퇴 안내 링크 또는 가이드 제공

## 5. 탐지 방법 (기술 메모)

### 가입 이메일 찾기

- **발신 도메인 클러스터링** — `noreply@`, `support@`, `notifications@` 등 발신 주소는 달라도 도메인 기준으로 묶으면 "서비스 단위" 식별이 됨
- **해당 도메인에서 받은 첫 메일 날짜** = 가입 시점 proxy
- **Gmail 자체 카테고리 분류 활용** — `category:updates`, `category:purchases`, `category:reservations`는 구글이 이미 계산해 둔 거래성 메일 분류라 언어에 무관하다. 1차 필터링 예: `from:(noreply OR no-reply) OR category:updates`
- **List-Unsubscribe 헤더** — RFC 표준 헤더라 언어에 상관없이 구독/커머셜 메일 성격을 잡아냄. 단 **검색 연산자로는 쓸 수 없고**(Gmail은 임의 헤더 검색을 지원하지 않음 — `rfc822msgid:`, `deliveredto:`만 예외), 메시지를 가져온 뒤 헤더로 조회해 분류에만 쓴다.

### 장기 미사용 계정 판단

MVP 기준 완전 자동화는 어렵다.

- 자동: 도메인별 마지막 수신일 기준으로 정렬 (오래된 순). 단 **광고성 메일은 이 계산에서 제외** — 방치된 계정도 뉴스레터는 계속 오기 때문
- 수동: Tinder식 스와이프로 "유지/삭제"를 사용자가 최종 확인

⇒ AI가 "이건 안 쓰는 것 같다"고 확신 있게 판단하는 기능보다, **정리 우선순위를 정렬해주고 확인은 사용자가 하게** 하는 편이 안전하고 빠르게 만들 수 있다.

### 왜 확장 프로그램을 쓰지 않는가

`chrome.history`로 방문 기록을 읽는 방향도 검토했으나 보류했다.

- 크롬 히스토리 창은 **약 90일**이다. 우리가 찾으려는 방치 계정(12~36개월+)은 정의상 그 안에 없다 — 히스토리로는 애초에 발견이 안 된다.
- 히스토리에 있다는 건 "최근 썼다"는 강한 신호이므로 **오탐 방지**에는 쓸모가 있지만, 없다고 방치의 증거는 되지 못한다.
- 설치 경고문이 *"모든 로그인된 기기에서 인터넷 사용 기록 읽기 및 변경"*이다. 읽기 전용 권한은 없다. 새 발견이 아니라 안전장치 하나를 얻자고 치르기엔 신뢰 비용이 크다.

## 6. 차별점

기존 서비스:

- 비밀번호 관리자 → 로그인 관리
- 개인정보 유출 확인 → 위험 확인
- 이메일 검색 → 사용자가 직접 탐색

**Digital Footprint Manager:**

> "내가 남긴 디지털 흔적을 찾아주고, 무엇을 정리해야 하는지 알려주는 서비스"

- 단순 보안이 아닌 **"디지털 정리"** 목적
- 발견 → 분석 → 삭제 행동까지 연결
- 과거에 쌓인 데이터를 관리하는 서비스
