# Digital Footprint Manager

> **"인터넷에 남긴 나의 흔적을 찾아 정리하는 디지털 청소 서비스"**

**바로 실행하기:** `prototype/run.cmd` 를 더블클릭하거나 `node prototype/run.mjs`

## 1. 문제의식

사람들은 수많은 온라인 서비스에 가입하지만, 시간이 지나면 자신의 디지털 흔적을 관리하지 못한다.

- 오래된 계정을 기억하지 못함
- 사용하지 않는 계정이 방치됨
- 어디에 내 개인정보가 남아 있는지 모름
- 지우고 싶어도 탈퇴 방법을 찾기 어려움

**핵심 문제**

> "내가 인터넷에 무엇을 남겼는지 모르고, 정리 방법도 어렵다."

## 2. Target User

### ① 20~30대 디지털 사용자

- SNS, 쇼핑몰, 커뮤니티 등 다양한 서비스 이용
- 오래된 계정 정리 필요

### ② 개인정보 보호 관심 사용자

- 불필요한 계정 삭제 및 데이터 관리 필요

## 3. Solution

### ① 계정 찾기

메일함에는 흔적이 남는다. 가입할 때 온 "환영합니다", 비밀번호를 잊었을 때 온 "재설정 안내",
결제했을 때 온 영수증. 이 메일들을 보면 **내가 어디에 가입했는지 알 수 있다.**

그래서 Gmail을 읽는다. 단, **보내는 사람과 제목만** 본다. 메일 본문은 읽지 않는다.

### ② 얼마나 확실한지 알려주기

찾았다고 다 계정인 건 아니다. 그래서 **점수와 근거를 같이 보여준다.**

| 신뢰 | 뜻 |
|---|---|
| 높음 | 가입 메일이나 비밀번호 재설정 메일이 있다. 계정이 있는 게 거의 확실하다 |
| 검토 | 정황은 있는데 단정하기 어렵다. 직접 확인이 필요하다 |
| 낮음 | 메일은 왔지만 계정인지는 모르겠다 |

`75점 · 가입 완료 + 비밀번호 재설정` 처럼 **왜 그 점수인지도 같이 적는다.** 우리 판단을 믿으라고
하지 않고, 근거를 보여주고 사용자가 판단하게 한다.

### ③ 탈퇴 방법 안내

계정을 찾아줘도 탈퇴 페이지를 못 찾으면 소용이 없다. 그래서 **한국 서비스 46곳의 탈퇴 경로를 직접
확인해서 정리해뒀다.** 쿠팡, 배달의민족, 토스, 당근, 무신사, 올리브영, 야놀자 같은 곳들이다.

각 서비스마다:

- 어디로 가서 뭘 눌러야 하는지 (실제 화면 경로)
- **탈퇴하기 전에 정리해야 하는 것** (토스는 잔액이 0원이어야 하고, 배민은 탈퇴해도 리뷰가 안 지워지고, YES24는 적립금이 사라진다)
- 취소가 되는지, 며칠 안에 되는지
- 개인정보 삭제 요청문 (복사해서 보내면 됨)

**대신 지워주지는 않는다.** 탈퇴는 각 서비스 공식 화면에서 사용자가 직접 한다.

## 4. 지금 동작하는 것

브라우저에서 설치 없이:

1. Google 계정으로 로그인
2. **Gmail 연결하고 스캔** 버튼을 누르면 그때 메일 권한을 따로 요청
3. 메일함을 훑으면서 서비스 목록이 실시간으로 쌓임
4. 서비스마다 **탈퇴** 버튼을 누르면 그 서비스의 탈퇴 안내가 열림

**실제 스캔 결과** (메일 811통 기준, 약 3분)

> 서비스 62곳 발견
>
> 사람이 보낸 메일 10건은 자동으로 제외 (본인 주소, 지인 메일)
>
> 제외가 잘못됐으면 **복구** 버튼으로 되돌릴 수 있음

## 5. 아직 없는 것

솔직하게 적는다.

- **메일 삭제.** 메일을 지우려면 구글이 "이 앱이 당신 이름으로 메일을 보낼 수 있게 허용"을
  같이 요구한다. 휴지통 전용 권한 같은 건 없다. 대신 Gmail 검색 링크로 보내줄 예정이다.
- **탈퇴 우선순위.** 지금은 "계정이 있느냐" 순으로 정렬한다. "먼저 치워야 하느냐" 순은 다음 작업이다.
- **유출 이력 확인.** 쓸 만한 데이터 출처가 없어서 뺐다.
- **소셜 로그인으로만 가입한 계정.** 메일이 안 오니까 못 찾는다. 이건 기술로 해결이 안 된다.
  목록에 없다고 계정이 없는 건 아니다.

## 6. 개인정보를 어떻게 다루나

이 제품의 핵심이고, 경쟁 서비스가 죽은 이유이기도 하다.

- **메일 본문을 읽지 않는다.** 보내는 사람과 제목 같은 헤더만 본다.
- **메일이 서버로 가지 않는다.** 스캔은 브라우저 안에서 끝난다. 우리 서버는 메일을 받지 않고,
  저장하지도 않는다.
- **메일 권한을 처음부터 요구하지 않는다.** 로그인과 메일 접근이 분리돼 있다.
- **권한은 언제든 끊을 수 있다.** [Google 계정 설정](https://myaccount.google.com/permissions)에서.

비슷한 서비스(Mine, Deseat.me, Jumbo)는 전부 죽거나 팔렸다. 수요가 없어서가 아니라 **개인정보를
지켜준다면서 메일함 전체를 서버로 가져가는 모순** 때문이었다. 그래서 브라우저 안에서 끝낸다.

## 7. 어떻게 찾는가 (기술 메모)

### 서비스 단위로 묶기

`noreply@`, `support@`, `notifications@` 처럼 발신 주소는 달라도 도메인으로 묶으면 서비스 하나가
된다. 단순히 `@` 뒤를 자르면 안 되고 실제 등록 도메인까지 줄여야 한다. `shop.example.co.kr` 은
`example.co.kr` 이지 `co.kr` 이 아니다. 한국 도메인이 다 이 구조라서 이걸 틀리면 전부 깨진다.

### 사람과 서비스 구분

`gmail.com`, `naver.com` 에서 온 메일은 대부분 사람이다. 그래서 개인 메일함 도메인은 주소 단위로
따로 본다. 다만 `noreply@gmail.com` 처럼 서비스가 Gmail로 보내는 경우도 있어서, 가입이나 인증
메일이 있으면 다시 살려낸다.

### 위조 메일 걸러내기

"가입을 환영합니다" 라고 쓰인 피싱 메일로 가짜 계정을 만들어낼 수 있다. 그래서 구글이 인증에
성공했다고 표시한 메일만 점수에 넣는다.

### 광고 메일 판별

방치된 계정도 뉴스레터는 계속 온다. 그래서 광고성 메일은 "마지막 흔적" 계산에서 뺀다. 판별에는
구글의 자체 분류와 메일 헤더(`List-Unsubscribe`, `List-Id` 등)를 같이 쓴다. 어느 하나만으로
단정하지 않는다. 영수증에도 수신거부 링크가 붙기 때문이다.

### 왜 확장 프로그램을 안 쓰나

방문 기록을 읽는 방향도 검토했지만 접었다.

- 크롬 히스토리는 **약 90일**만 남는다. 우리가 찾는 방치 계정은 1년 이상 안 쓴 것들이라
  **정의상 히스토리에 없다.**
- 설치 경고문이 *"모든 로그인된 기기에서 인터넷 사용 기록 읽기 및 변경"* 이다. 읽기 전용 권한이
  없다. 새로 찾아내는 것도 없는데 치르기엔 신뢰 비용이 너무 크다.

## 8. 차별점

| 기존 | 하는 일 |
|---|---|
| 비밀번호 관리자 | 로그인 관리 |
| 유출 확인 서비스 | 위험 알림 |
| 이메일 검색 | 사용자가 직접 찾아야 함 |

**Digital Footprint Manager**

> "내가 남긴 디지털 흔적을 찾아주고, 무엇을 어떻게 정리해야 하는지 알려주는 서비스"

- 보안이 아니라 **디지털 정리**가 목적이다
- 찾기, 판단, 탈퇴 안내까지 이어진다
- 살아 있는 경쟁자 중 **브라우저 안에서만 처리하는 곳은 없다**

## 9. 개발자용

```
prototype/
  frontend/    브라우저에서 도는 전부 (스캔, 분류, 점수, 화면)
  backend/     로그인과 설정만. 메일은 만지지 않음
  test/        유닛 118개 + e2e 15개
docs/tracking/findings.md   알고 있지만 아직 안 고친 것들
PRODUCT_SPEC.md             제품 계약서. §8에 결정 기록
```

```bash
cd prototype
npm install
cp .env.example .env     # GOOGLE_CLIENT_ID 채우기
npm test                 # 유닛
npm run test:e2e         # 브라우저 (가짜 Gmail로 실제 앱 구동)
node run.mjs             # 서버 + 브라우저 열기
```

Google Cloud Console에서 **승인된 JavaScript 원본**에 `http://localhost:3456` 을 등록해야 로그인
버튼이 동작한다.
